\section{Introduction}
Final Year Project (FYP) is an extremely significant and mandatory component of undergraduate degree program. The stage involves students combining practical and theoretical knowledge to solve a real world problem. It is essentially the final substantial need prior to the achievement of the degree thus has much significance to the students and the university.
It is not so easy to manage the FYP process at any University. Today all the FYP related work is performed manually. Students would be forced to hand in their ideas on paper and wait to have them approved by the coordinator. They do not have an appropriate follow-up of what group is at what point. University mostly stores FYP data in Excel sheets and other manual tools to track the project development process since the group formation, to the final evaluation.
This manual method introduces numerous issues such as repetition of data in various files, the maintenance of the information is not done in the correct manner and the entire process is slow and ineffective. This approach is becoming more difficult to cope with as the amount of students is rising annually. We are designing an FYP Management System, which is specifically developed to address all these issues and is to be applied to Namal University. Any FYPMS needs a centralized database  driven to digitalize this system and automate all the stages of FYP like proposal submission, supervisor assignment and final submission to ensure that all processes are done in a smooth manner through a single centralized location. A well designed database system can defines the system automation and these can settle the FYPMS with modern digital academics plateform.

\section{Problem Statement}


All the tasks associated with Final Year Projects are now handled at Namal University using manual processes or basic Excel based programs. The management team with coordinators and faculty are struggling with numerous challenges as a consequence of the rising number of students and projects per year. There is no centralized platform that to manage FYP process from student group formation to last final defense that creates data redundancy, inconsistency, loss of records and significant administrative overhead.
 All these problems will be resolved by our proposed FYP Management System which will offer one platform where all steps will be actively handled and monitored with relational database driven system that centralizes all FYP tasks,data integrity and role based access for every users. Reduce the gap between data and system which gives smooth FYP Management system.

\section{Objectives}
The key aims of the FYP Management System include:

\begin{enumerate}
    \item To computerize the entire process of FYP management that involves formation of student groups up to the last assessment.


    \item To grant role based access to all users such that the students, supervisors, coordinators  can only view and perform what is relevant to their position.



    \item To have adequate and comprehensive records of all FYPs of the past and present that are kept in one centralized system.
   
    \item To design and implement a structured relational database to efficiently store and handle FYP-related data.
  
    \item To eliminate the duplication of data and have all information uniform and stored under one roof.

    \item To ensure that the system is scalable such that it can support a high number of users and projects without any performance problems.

    \item Make ensure data integrity, consistency and security in system database by using  constraints, normalization and access control mechanisms.

 
\end{enumerate}

\section{Scope}

A centralized web application to digitize and handle the complete Final Year Project lifecycle at Namal University.

\subsection{Included Features}

    \begin{itemize}
        \item \textbf{Secure login} with role-based access control.
        
        \item \textbf{Centralized relational database} for all FYP data.
        
        \item \textbf{Management of student }groups, projects, and advisor assignments.

    \item \textbf{No AI-based features} System has no AI integration init.
    
    \item \textbf{No non-FYP processes} There is external processes like attendance, course registration.
    
\end{itemize}

\section{System Functionality}

The Final Year Project Management System (FYPMS) gives a centralized, specific role-based 
platform that handles all phases of FYPs. System has 
5 different users that has functionalities based on their role and responsibilities. All users has same authentication method, but tasks and functionalities are differ by role to ensure data security and access control.

\subsection{Common Functionality}

\textbf{Authentication} is a functionality that is the same across all users. Every user logs in using stored credentials table and checks their data related to Person record .After login, system knows the user's role and gives access to features and data specific to that role only.

\subsection{Coordinator}

The Coordinator is a user responsible for handling all FYP related data and its workflow. Coordinator performs the following functions:

\begin{itemize}
    \item \textbf{Group Management} that gives access to create, update and delete student groups.
    
    \item \textbf{Project Management} has ability to create, update and handle project data.
    
    \item \textbf{Project Assignment Management} assigns a project to a specific student group.
    
    \item \textbf{Advisor Assignment Management} allows linking a supervisor and co-supervisor to each project assignment.
    
    \item \textbf{Evaluation Management}   defines and manages evaluation milestones.
    
    \item \textbf{Evaluation Result Management} records and updates group evaluation results.
\end{itemize}

\subsection{Super Admin}

The Super Admin works at an administrative level and does not interact with FYP content directly. Super Admin responsibilities  on user management and access control.

\begin{itemize}
    
    \item \textbf{User Account Management} allows creating, updating, and deleting credentials,user records and assigning roles to each user account.
    
    \item \textbf{Student, Faculty and Evaluator Account Management} enables to create and handle student, faculty, and evaluator records.
    
    \item \textbf{Role and Permission Management} assigns and modifies user roles across all account types.
    
    \item \textbf{Credential Management} has ability to reset and update username and password for any user account when needed.
    
    \item \textbf{User Record Viewing} allows to view all registered persons and their associated account roles across the system.
    
\end{itemize}

\subsection{Student}

Student has ability to view information relevant to its group and project.

\begin{itemize}
    \item \textbf{Group Information Viewing} allows to view assigned group details.
    
    \item \textbf{Project Information Viewing} allows to view the project details.
    
    \item \textbf{Advisor Information Viewing} enables to view the details of the supervisor and co-supervisor associated to their group's project.
    
    \item \textbf{Evaluation Result Viewing} allows to view  evaluation results.
     
    \item \textbf{Adivisor comments and status of Document/text of milestone  Viewing} allows to view  feedback or comment and status of the milestone submission.

  \item \textbf{Note:} There are two option to store pdf or files, ones is stores the path of file in database and other we make tables related to type of proposal to info directly tables.
    
\end{itemize}

\subsection{Faculty (Supervisor / Co-Supervisor)}

Faculty members can  monitor their assigned groups and provide 
academic guidance during the FYP phase.

\begin{itemize}
    \item \textbf{Assigned Group Viewing} allows to view all groups where appear as supervisor or co-supervisor.
    
    \item \textbf{Group Membership Viewing} allows to view  students of each assigned group.
    
    \item \textbf{Project Detail Viewing} provides access to view projects details associated to their assigned groups.
    
    \item \textbf{Evaluation Schedule Viewing} allows  to view  evaluation details.
    
    \item \textbf{Feedback and Comment Submission} allows to submit  feedback and comments on project submissions.
\end{itemize}

\subsection{Evaluator}

Evaluators are those check the different milestones of the FYP. These are following functionalities:

\begin{itemize}
    \item \textbf{Assigned Group Viewing} allows to view the groups assigned for evaluation.
    
    \item \textbf{Project Detail Viewing} enables to view the project details of their assigned groups.
    
    \item \textbf{Evaluation Criteria Viewing}  view the evaluation criteria.
    
    \item \textbf{Evaluation Result Submission} allows to insert evaluation result and comment into the Evaluation Result table.
\end{itemize}

\section{Conclusion}

Final Year Project Management System represents advancement in term how academic project management is managed at Namal University and 
current methodology is manual, separated in multiple processes that creates a inefficiencies.These
affect the users and administrative . designing a system with centraliazed
relational database present in the foundation of FYP process. This proposed FYPMS will provide
a platform that is  technically and practically useful. Implementation of this system can enhance efficiency, reduce manual workload and maintain data records which work as centralized with the help of a well-designed database and web interface. The FYP management process can become more transparent, reliable and easier to handle.
